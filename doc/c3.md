# Capítulo 3 — Implementación de la aplicación

## 3.1. Visión general de los endpoints

La aplicación está dividida en dos bloques principales: un backend desarrollado con Node.js y Express, y un frontend desarrollado con ReactJS.

El backend mantiene la arquitectura implementada durante M1 basada en rutas, controladores y middlewares. Por su parte, el frontend proporciona una interfaz gráfica que permite interactuar con la API REST mediante formularios y componentes React.

La comunicación entre ambos componentes se realiza mediante peticiones HTTP utilizando representaciones JSON.

## 3.2. Frontend React

La aplicación cliente se encuentra en el directorio `frontend/` y está desarrollada utilizando ReactJS.

Los componentes principales implementados son:

1. `Login.js` — Gestiona la autenticación de usuarios.
2. `Profile.js` — Muestra la información del usuario autenticado.
3. `Projects.js` — Permite gestionar proyectos.
4. `Tasks.js` — Permite gestionar tareas.
5. `App.js` — Coordina la navegación y el estado global de la aplicación.

Cada componente es responsable de una funcionalidad concreta, facilitando el mantenimiento y la reutilización del código.

## 3.3. Gestión de la autenticación

La autenticación se realiza mediante un formulario de login implementado en el componente `Login.js`.

El usuario introduce su dirección de correo electrónico y contraseña. Para ello se utilizan estados de React que almacenan los valores introducidos en los campos del formulario:

```javascript
const [email, setEmail] = useState("");
const [password, setPassword] = useState("");
```

Cuando el usuario envía el formulario, se ejecuta una función que realiza la petición al endpoint `POST /auth/login`:

```javascript
async function handleSubmit(event) {
  event.preventDefault();

  const data = await login(email, password);

  if (data.token) {
    saveToken(data.token);
    onLogin();
  } else {
    alert("Login incorrecto");
  }
}
```

La función `login()` realiza la llamada HTTP a la API REST:

```javascript
export async function login(email, password) {
  const response = await fetch(API_URL + "/auth/login", {
    method: "POST",
    headers: {
      "Content-Type": "application/json"
    },
    body: JSON.stringify({ email, password })
  });

  return response.json();
}
```

Si las credenciales son válidas, la API devuelve un token JWT que se almacena en `localStorage`:

```javascript
export function saveToken(token) {
  localStorage.setItem("token", token);
}
```

A partir de ese momento, el token se incluye automáticamente en todas las peticiones protegidas mediante la cabecera:

```javascript
Authorization: Bearer <token>
```

Si el usuario cierra sesión, el token se elimina de `localStorage`:

```javascript
export function removeToken() {
  localStorage.removeItem("token");
}
```

Este mecanismo permite mantener la sesión iniciada mientras el token permanezca almacenado en el navegador.


## 3.4. Consulta del perfil de usuario 

El componente `Profile.js` permite visualizar la información del usuario autenticado.

Para ello realiza una petición al endpoint `GET /users/me` utilizando el token JWT almacenado previamente.

La función encargada de realizar esta consulta se encuentra en `api.js`:

```javascript
export async function getUser() {
  const response = await fetch(API_URL + "/users/me", {
    headers: {
      Authorization: "Bearer " + getToken()
    }
  });

  return response.json();
}
```

Cuando el componente se carga, React ejecuta automáticamente una petición utilizando el hook `useEffect()`:

```javascript
useEffect(() => {
  async function loadUser() {
    const data = await getUser();
    setUser(data);
  }

  loadUser();
}, []);
```

La información recibida se almacena en un estado local:

```javascript
const [user, setUser] = useState(null);
```

Finalmente, los datos se muestran en pantalla:

```javascript
<p>
  <strong>Username:</strong> {user.username}
</p>

<p>
  <strong>Email:</strong> {user.email}
</p>
```

Esta funcionalidad permite comprobar que el usuario ha sido autenticado correctamente y que el token JWT sigue siendo válido.

## 3.5. Gestión de proyectos

El componente `Projects.js` implementa las operaciones CRUD sobre los proyectos del usuario autenticado.

Las funcionalidades disponibles son:

1. Consultar proyectos existentes.
2. Crear nuevos proyectos.
3. Modificar proyectos existentes.
4. Eliminar proyectos.

Para consultar los proyectos se utiliza el endpoint `GET /projects`.

La función utilizada para obtener los datos es la siguiente:

```javascript
export async function getProjects() {
  const response = await fetch(API_URL + "/projects", {
    headers: {
      Authorization: "Bearer " + getToken()
    }
  });

  return response.json();
}
```

La carga inicial de proyectos se realiza utilizando `useEffect()`:

```javascript
useEffect(() => {
  async function loadProjects() {
    const data = await getProjects();

    if (Array.isArray(data)) {
      setProjects(data);
    } else {
      setProjects([]);
    }
  }

  loadProjects();
}, []);
```

Para crear un nuevo proyecto se utiliza el endpoint `POST /projects`:

```javascript
await createProject(name, description);
```

La función correspondiente es:

```javascript
export async function createProject(name, description) {
  const response = await fetch(API_URL + "/projects", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: "Bearer " + getToken()
    },
    body: JSON.stringify({ name, description })
  });

  return response.json();
}
```

La actualización de proyectos se realiza mediante el endpoint `PUT /projects/:id`:

```javascript
await updateProject(id, name, description);
```

Mientras que la eliminación utiliza el endpoint `DELETE /projects/:id`:

```javascript
await deleteProject(id);
```

Todos los proyectos mostrados pertenecen exclusivamente al usuario autenticado, ya que la API filtra automáticamente los recursos utilizando la identidad contenida en el token JWT.

## 3.6. Gestión de tareas

El componente `Tasks.js` implementa las operaciones CRUD sobre las tareas asociadas a los proyectos del usuario.

Las funcionalidades disponibles son:

1. Consultar tareas existentes.
2. Crear nuevas tareas.
3. Modificar tareas existentes.
4. Eliminar tareas.

La consulta de tareas utiliza el endpoint `GET /tasks`:

```javascript
export async function getTasks() {
  const response = await fetch(API_URL + "/tasks", {
    headers: {
      Authorization: "Bearer " + getToken()
    }
  });

  return response.json();
}
```

Al igual que en el componente de proyectos, la carga inicial se realiza mediante `useEffect()`:

```javascript
useEffect(() => {
  async function loadTasks() {
    const data = await getTasks();

    if (Array.isArray(data)) {
      setTasks(data);
    } else {
      setTasks([]);
    }
  }

  loadTasks();
}, []);
```

La creación de tareas se realiza utilizando el endpoint `POST /tasks`:

```javascript
await createTask(title, description, false, Number(projectId));
```

La función correspondiente es:

```javascript
export async function createTask(title, description, completed, project_id) {
  const response = await fetch(API_URL + "/tasks", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: "Bearer " + getToken()
    },
    body: JSON.stringify({
      title,
      description,
      completed,
      project_id
    })
  });

  return response.json();
}
```

La actualización de tareas utiliza el endpoint `PUT /tasks/:id`:

```javascript
await updateTask(id, title, description, completed);
```

La eliminación se realiza mediante:

```javascript
await deleteTask(id);
```

La API verifica que el usuario autenticado sea propietario del proyecto asociado antes de permitir la creación, modificación o eliminación de tareas.

Este mecanismo garantiza el aislamiento de recursos y evita que un usuario pueda manipular información perteneciente a otros usuarios.

## 3.7. Navegación de la aplicación

La navegación se implementa desde el componente principal `App.js`.

Una vez autenticado, el usuario puede acceder a las distintas secciones mediante botones de navegación:

1. User Profile.
2. Projects.
3. Tasks.
4. Logout.

El cambio entre vistas se realiza mediante el estado interno de React, sin necesidad de recargar la página.

## 3.8. Gestión del estado

La aplicación utiliza los hooks proporcionados por React para gestionar el estado y el ciclo de vida de los componentes.

Los hooks utilizados son:

1. `useState()` para almacenar información dinámica.
2. `useEffect()` para ejecutar peticiones a la API cuando se carga un componente.

Esta aproximación simplifica la gestión de datos y mantiene sincronizada la interfaz con la información almacenada en la API.

## 3.9. Dockerización del frontend

La aplicación React ha sido dockerizada mediante un Dockerfile específico ubicado en el directorio `frontend/`.

El proceso seguido ha sido:

1. Construcción de la imagen Docker del frontend.
2. Publicación de la imagen en Docker Hub.
3. Integración de la imagen dentro de `docker-compose.yml`.
4. Despliegue conjunto junto con la API y la base de datos.

La imagen publicada utiliza el repositorio `inf25dw2g05/taskmanager-frontend`.

## 3.10. Integración completa del sistema

La combinación del frontend React, la API REST y la base de datos MySQL permite disponer de una aplicación web completamente funcional.

El usuario puede autenticarse, consultar su perfil, gestionar proyectos y administrar tareas utilizando una única interfaz web.

Todas las operaciones realizadas desde React son procesadas por la API REST, que aplica los mecanismos de autenticación, autorización y control de acceso desarrollados durante la primera fase del proyecto.
