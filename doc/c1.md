# Capítulo 1 — Introducción

## 1.1. Descripción del proyecto

**TaskManager Web Application** es una aplicación REST desarrollada en el marco de la asignatura *Desarrollo Web II* (M2) del grado en Informática de la UMAIA, curso 2025/2026.

El proyecto reutiliza la API REST desarrollada en la primera evaluación continua y añade una aplicación cliente desarrollada con ReactJS para consumir los recursos ofrecidos por dicha API.

El sistema permite a los usuarios gestionar proyectos personales y las tareas asociadas a esos proyectos mediante una interfaz web. Cada usuario dispone de un espacio aislado: únicamente puede acceder, crear, modificar y eliminar sus propios proyectos y tareas. Las credenciales y la autorización continúan gestionándose mediante tokens JWT.

La solución está compuesta por tres componentes principales: 

- Una base de datos **MySQL** para el almacenamiento persistente de la    información.
- Una **API REST** desarrollada con **Node.js** y **Express**.
- Una aplicación cliente desarrollada con **ReactJS**.

Los tres componentes se despliegan mediante Docker Compose.

## 1.2. Objetivos

El proyecto cumple con los requisitos obligatorios del enunciado:

- Arquitectura **REST**, utilizando los cuatro verbos HTTP (GET, POST, PUT, DELETE).
- **Tres recursos** distintos: `users`, `projects` y `tasks`.
- **Dos relaciones 1:n** entre los recursos: `users → projects` y `projects → tasks`.
- Representaciones en **JSON**.
- **Autenticación y autorización** mediante JWT, con control de acceso por propietario.
- **Logging** del usuario autenticado en consola para cada petición protegida.
- Documentación **OpenAPI 3.0** servida vía Swagger UI.
- **Colección Postman** para consulta de los recursos.
- Despliegue **multi-contenedor** con Docker Compose (MySQL + Node.js).
- **Base de datos MySQL** poblada con al menos 30 registros por tabla.

Adicionalmente, se han incorporado los siguientes elementos valorados en el enunciado:

- Aplicación cliente desarrollada con **ReactJS**.
- Gestión de autenticación mediante **JWT** desde la interfaz web.
- Consulta del perfil del usuario autenticado.
- Operaciones **CRUD** sobre proyectos desde la aplicación React.
- Operaciones **CRUD** sobre tareas desde la aplicación React.
- Navegación simple entre las secciones **User Profile**, **Projects** y **Tasks**.
- Dockerización completa del frontend React.
- Publicación de imágenes Docker en **Docker Hub**.
- Despliegue **multi-contenedor** con Docker Compose (**MySQL + API + Frontend React**).

## 1.3. Stack tecnológico

| Capa | Tecnología |
|---|---|
| Frontend | React JS |
| Servidor aplicacional | Node.js 20 (Alpine) |
| Framework web | Express 5 |
| Base de datos | MySQL 8.0 |
| Driver MySQL | mysql2 (con pool de conexiones) |
| Autenticación | JSON Web Tokens (jsonwebtoken) |
| Hash de contraseñas | bcryptjs |
| Documentación API | OpenAPI 3.0 + swagger-ui-express |
| Contenerización | Docker + Docker Compose |
| Variables de entorno | dotenv |
| CORS | cors |
