# Capítulo 4 — Autenticación y autorización

## 4.1. Conceptos previos

La autenticación es el proceso mediante el cual una aplicación verifica la identidad de un usuario. Una vez autenticado, el sistema puede determinar qué recursos puede consultar o modificar.

Por su parte, la autorización consiste en comprobar si un usuario autenticado dispone de permisos para realizar una determinada operación sobre un recurso concreto.

En el proyecto TaskManager ambos mecanismos se implementan utilizando JSON Web Tokens (JWT).

## 4.2. JSON Web Tokens (JWT)

Un JSON Web Token (JWT) es un estándar abierto utilizado para transmitir información de forma segura entre dos partes mediante un objeto JSON firmado digitalmente.

El token generado por la API contiene información sobre la identidad del usuario autenticado y permite evitar el envío continuo de credenciales en cada petición.

El flujo general de funcionamiento es el siguiente:

1. El usuario introduce sus credenciales.
2. La API valida el correo electrónico y la contraseña.
3. La API genera un JWT firmado.
4. El frontend almacena el token.
5. Las peticiones posteriores utilizan dicho token para identificarse.

Esta aproximación permite implementar autenticación sin estado (*stateless authentication*), ya que el servidor no necesita mantener sesiones activas.

## 4.3. Flujo de autenticación en la aplicación

El proceso completo de autenticación implementado en la aplicación puede resumirse de la siguiente forma:

Usuario
↓
Login.js
↓
POST /auth/login
↓
API REST
↓
JWT
↓
localStorage
↓
Peticiones protegidas

Cuando el usuario accede a la aplicación, introduce sus credenciales en el formulario de login.

El frontend envía dichas credenciales al endpoint `POST /auth/login`.

Si la autenticación es correcta, la API devuelve un token JWT que se almacena en `localStorage`.

A partir de ese momento, todas las peticiones protegidas incorporan la cabecera de autorización:

`Authorization: Bearer <token>`

## 4.4. Verificación del token

La API incorpora un middleware encargado de validar el JWT recibido en cada petición protegida.

Antes de ejecutar cualquier controlador, el middleware comprueba:

1. Que la cabecera Authorization existe.
2. Que el token tiene un formato válido.
3. Que la firma del token es correcta.
4. Que el token no ha expirado.

Un ejemplo simplificado del middleware utilizado es el siguiente:

```javascript
const token = authHeader.split(" ")[1];

const decoded = jwt.verify(token, process.env.JWT_SECRET);

req.user = decoded;
